Mô tả quan hệ:

  • USER - POST (1:N): Một user có thể tạo nhiều bài đăng
  • USER - COMMENT (1:N): Một user có thể viết nhiều bình luận
  • POST - COMMENT (1:N): Một bài đăng có thể có nhiều bình luận
  • USER - LIKE (1:N): Một user có thể like nhiều bài đăng/bình luận
  • POST/COMMENT - LIKE (1:N): Một bài đăng/bình luận có thể được like nhiều lần
  • USER - STORY (1:1): Một user chỉ có thể có một story tại một thời điểm
  • USER - FOLLOWER (M:N): Quan hệ nhiều-nhiều giữa người theo dõi và được theo dõi
  • USER - FOLLOW_REQUEST (M:N): Quan hệ nhiều-nhiều cho yêu cầu follow
  • USER - BLOCK (M:N): Quan hệ nhiều-nhiều cho việc chặn người dùng
